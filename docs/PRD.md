# PDFConvert 产品需求文档 (PRD)

**版本**: v2.0
**日期**: 2026-02-27
**状态**: 草案

---

## 1. 项目概述

### 1.1 背景

PDFConvert 是一个 PDF 转 Markdown 的命令行工具。当前版本 (v1.0) 基于 PaddleOCR 实现纯文字 OCR 提取，存在以下局限：

- 无法识别数学公式并转为 LaTeX
- 无法理解图表/图标内容
- 无法保留文档结构（标题层级、列表、表格）
- 对手写内容识别能力有限

### 1.2 目标

升级为基于本地视觉大模型 (GLM-4.6V-Flash) 的方案，通过 LM Studio 本地部署，实现：

- 高质量的文档结构识别与保留
- 数学公式 -> LaTeX 转换
- 图表内容理解与数据提取
- 全流程本地运行，无需联网，保护数据隐私

### 1.3 技术环境

| 项目 | 规格 |
|------|------|
| 语言 | Python 3.12 |
| 包管理 | uv |
| 模型 | GLM-4.6V-Flash (9B) |
| 模型服务 | LM Studio (端口 1234, OpenAI 兼容 API) |
| 平台 | macOS Apple Silicon |

## 2. 用户故事

### US-1: 基本 PDF 转换
**作为** 一名学生/研究人员，**我希望** 将 PDF 文档转换为 Markdown 格式，**以便** 在笔记工具中编辑和引用。

**验收标准**:
- 输入一个 PDF 文件路径，输出对应的 .md 文件
- 支持中英文混合文档
- 保留原文档的标题层级 (h1-h6)、列表、表格结构

### US-2: 数学公式识别
**作为** 一名理科学生，**我希望** PDF 中的数学公式被转换为 LaTeX 格式，**以便** 在 Markdown 中正确渲染。

**验收标准**:
- 行内公式用 `$...$` 包裹
- 独立公式块用 `$$...$$` 包裹
- 常见数学符号（积分、求和、分数、矩阵等）正确转换

### US-3: 图表理解
**作为** 用户，**我希望** PDF 中的图表被识别并转为文字描述，**以便** 在纯文本环境中理解图表内容。

**验收标准**:
- 图表生成文字描述，说明图表类型和主要内容
- 尽可能提取图表中的关键数据点
- 描述以引用块 (`>`) 格式嵌入 Markdown

### US-4: 自动模型管理
**作为** 用户，**我希望** 工具能自动检测并加载所需模型，**以便** 无需手动操作 LM Studio。

**验收标准**:
- 启动时检测 LM Studio 是否运行
- 检测 GLM-4.6V-Flash 是否已加载
- 未加载时通过 `lms` CLI 自动加载
- 加载失败时给出清晰的错误提示

## 3. 功能需求

### 3.1 模型管理模块

**F-1: LM Studio 连接检测**
- 启动时向 `http://localhost:1234/v1/models` 发送请求
- 超时时间: 5 秒
- 失败时提示用户启动 LM Studio

**F-2: 模型加载检测与自动加载**
- 从 `/v1/models` 响应中检查是否包含 GLM-4.6V-Flash
- 未加载时执行 `lms load glm-4v-flash-9b` (或对应模型标识)
- 加载后轮询确认模型就绪，超时 120 秒
- 加载失败时提示用户手动操作

### 3.2 PDF 处理模块

**F-3: PDF 转图片**
- 使用 `pdf2image` 将 PDF 每页转为 PNG 图片
- DPI: 300（平衡质量与处理速度）
- 图片以 base64 编码传入模型 API

**F-4: 逐页视觉识别**
- 调用 LM Studio OpenAI 兼容 API (`/v1/chat/completions`)
- 每页发送一张图片 + 结构化 prompt
- Prompt 指导模型识别内容类型并按规范格式输出

### 3.3 内容识别与转换

**F-5: 文字提取**
- 识别并提取页面中的纯文字内容
- 保留标题层级: `#` ~ `######`
- 保留有序/无序列表结构
- 保留表格结构 (Markdown 表格语法)
- 保留段落分隔

**F-6: 数学公式转换**
- 识别行内公式，输出为 `$LaTeX$`
- 识别独立公式块，输出为 `$$LaTeX$$`
- 支持常见数学符号: 分数、积分、求和、极限、矩阵、希腊字母等

**F-7: 图表描述**
- 识别图表类型（柱状图、折线图、饼图、流程图等）
- 生成自然语言描述
- 提取可见的数据标签和数值
- 输出格式:

```markdown
> **[图表类型]: [图表标题]**
> [图表描述]
> 关键数据: [提取的数据点]
```

### 3.4 输出模块

**F-8: Markdown 文件生成**
- 将所有页面内容合并为单个 Markdown 文件
- 页面之间用 `---` 分隔，并标注页码
- 文件编码: UTF-8
- 默认输出路径: 与输入 PDF 同目录，扩展名改为 `.md`

**F-9: 进度显示**
- 显示当前处理页码 / 总页数
- 显示每页的处理状态（识别中 / 完成）

## 4. 功能边界（Scope）

### 4.1 包含 (In Scope)

- 单个 PDF 文件转换为单个 Markdown 文件
- 中文、英文及中英混合文档
- 文字、数学公式、图表三种内容类型的识别
- 通过 LM Studio 本地 API 调用视觉模型
- 命令行界面 (CLI)

### 4.2 不包含 (Out of Scope)

- GUI 图形界面
- 批量 PDF 处理（可作为后续版本）
- 扫描件/手写体的高精度识别（依赖模型能力，不做额外优化）
- PDF 中嵌入的音视频内容
- 加密/受密码保护的 PDF
- 云端模型 API 调用
- PDF 表单 (Form) 字段提取
- 输出格式除 Markdown 外的其他格式（如 docx, html）

## 5. 输入输出规范

### 5.1 输入

| 参数 | 说明 | 必填 | 默认值 |
|------|------|------|--------|
| `pdf_path` | PDF 文件路径 | 是 | - |
| `output_path` | 输出 Markdown 文件路径 | 否 | `{pdf_name}.md` |

**CLI 用法**:
```bash
uv run python main.py <pdf_path> [output_path]
```

**输入约束**:
- PDF 文件大小: 不超过 200MB
- PDF 页数: 不超过 500 页
- 文件格式: 标准 PDF (ISO 32000)

### 5.2 输出

**Markdown 文件结构**:
```markdown
# [文档标题（如果可识别）]

---
<!-- Page 1 -->

[第1页内容: 文字/公式/图表描述]

---
<!-- Page 2 -->

[第2页内容]

...
```

**内容格式规范**:

| 内容类型 | 输出格式 |
|----------|----------|
| 标题 | `# H1` ~ `###### H6` |
| 段落 | 普通文本，段落间空行分隔 |
| 有序列表 | `1. item` |
| 无序列表 | `- item` |
| 表格 | Markdown 表格语法 |
| 行内公式 | `$E = mc^2$` |
| 独立公式 | `$$\int_0^1 f(x)dx$$` |
| 图表 | `> **[类型]: [标题]**\n> [描述]` |
| 页面分隔 | `---` + HTML 注释标注页码 |

## 6. 错误处理策略

### 6.1 错误分级

| 级别 | 说明 | 处理方式 |
|------|------|----------|
| FATAL | 无法继续运行 | 打印错误信息，退出程序 (exit code 1) |
| ERROR | 单页处理失败 | 跳过该页，在输出中标注，继续处理后续页 |
| WARN | 非关键问题 | 打印警告，继续运行 |

### 6.2 具体错误场景

| 场景 | 级别 | 处理方式 |
|------|------|----------|
| PDF 文件不存在 | FATAL | 提示 "文件不存在: {path}" |
| PDF 文件损坏/无法读取 | FATAL | 提示 "无法读取 PDF 文件，请检查文件是否损坏" |
| LM Studio 未启动 | FATAL | 提示 "无法连接 LM Studio，请确认已启动 (端口 1234)" |
| 模型加载失败 | FATAL | 提示 "模型加载失败，请在 LM Studio 中手动加载 GLM-4.6V-Flash" |
| 模型加载超时 (>120s) | FATAL | 提示 "模型加载超时，请检查系统资源" |
| 单页图片转换失败 | ERROR | 在输出中标注 `<!-- Page N: 转换失败 -->`，继续下一页 |
| 单页模型调用失败 | ERROR | 重试 1 次；仍失败则标注跳过，继续下一页 |
| 模型返回空内容 | WARN | 在输出中标注 `<!-- Page N: 未识别到内容 -->` |
| 输出文件写入失败 | FATAL | 提示 "无法写入输出文件: {path}，请检查权限" |

### 6.3 退出码

| 退出码 | 含义 |
|--------|------|
| 0 | 成功完成 |
| 1 | 致命错误，无法完成转换 |
| 2 | 部分完成（有页面处理失败） |

## 7. 非功能需求

### 7.1 性能

| 指标 | 目标 |
|------|------|
| 单页处理时间 | < 30 秒 (Apple Silicon, 9B 模型) |
| 内存占用 | 峰值 < 8GB (不含模型本身) |
| PDF 转图片 | < 2 秒/页 (300 DPI) |
| 模型加载时间 | < 120 秒 |

### 7.2 准确率目标

| 内容类型 | 目标准确率 | 说明 |
|----------|-----------|------|
| 印刷体文字 | >= 95% | 字符级准确率 |
| 文档结构 | >= 90% | 标题层级、列表、表格识别正确率 |
| 数学公式 | >= 80% | LaTeX 可正确渲染的比例 |
| 图表描述 | >= 70% | 图表类型识别正确 + 关键数据提取 |

> 注: 准确率受模型能力限制，以上为期望目标，实际表现需通过测试验证。

### 7.3 可靠性

- 单页失败不影响整体转换流程
- 模型调用失败自动重试 1 次
- 程序异常退出时不产生损坏的输出文件（先写临时文件，完成后重命名）

### 7.4 可维护性

- 模型名称、API 地址、DPI 等参数可通过配置或命令行参数调整
- 代码模块化: 模型管理、PDF 处理、内容转换、输出生成各为独立模块
- 日志输出到 stderr，不干扰 stdout 的正常输出

### 7.5 安全与隐私

- 全部处理在本地完成，不发送任何数据到外部服务器
- 不缓存或存储用户 PDF 的中间图片（处理完即释放内存）
